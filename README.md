# MinecraftContainerScanner

Сканирует регионы (`.mca`) Minecraft **1.18+**, находит контейнеры внутри заданных
кубоидов и выгружает их содержимое в JSON — чтобы потом показывать количество
ресурсов на сайте.

## Возможности

- Несколько **миров** — у каждого свой каталог регионов и свой набор кубоидов
  (имена `r.X.Z.mca` в разных мирах могут совпадать).
- Несколько **кубоидов** с именами; контейнеры фильтруются по координатам.
- Любые контейнеры (сундуки, бочки, шалкеры, воронки, печи и т.д.) — по наличию `Items`.
- **Рекурсия в шалкеры**: предметы внутри шалкеров тоже учитываются.
- Подсчёт **полных шалкеров**: если шалкер заполнен одним видом предмета и
  количество = `1728` / `864`, он засчитывается как полный шалкер этого предмета.
- Поддержка форматов предметов до и после 1.20.5 (`tag.BlockEntityTag` и `components`).

## Установка

```bash
npm install
```

## Настройка

Правится в [`config.js`](config.js):

```js
export const worlds = [
  {
    name: "world",
    regionDir: "./worlds/world/region",
    dimensions: [
      {
        name: "Склад",
        pos1: { x: 100, y: -60, z: 100 },
        pos2: { x: 140, y: 20, z: 140 },
      },
    ],
  },
];
```

`pos1`/`pos2` — два противоположных угла кубоида (порядок не важен).

## Запуск

```bash
node index.js
```

Результат — `containers.json` со структурой: миры → кубоиды → контейнеры,
плюс агрегаты `totals` и `fullShulkers` на каждом уровне и общие `grandTotals` /
`grandFullShulkers`.

## Структура

- [`index.js`](index.js) — обход миров/кубоидов, сбор предметов, полные шалкеры.
- [`region.js`](region.js) — чтение Anvil-формата `.mca` (заголовок + распаковка чанков).
- [`config.js`](config.js) — миры, кубоиды, пути.
