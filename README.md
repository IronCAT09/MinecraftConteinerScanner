# MinecraftContainerScanner

Сканирует регионы (`.mca`) Minecraft **1.18+**, находит контейнеры внутри заданных
кубоидов и выгружает их содержимое в JSON — чтобы потом показывать количество
ресурсов на сайте.

## Возможности

- Несколько **миров** — у каждого свой каталог регионов и свой набор кубоидов
  (имена `r.X.Z.mca` в разных мирах могут совпадать).
- Несколько **кубоидов** с именами; контейнеры фильтруются по координатам.
- Любые контейнеры (сундуки, бочки, шалкеры, воронки, печи и т.д.) — по наличию `Items`.
- **Рекурсия в шалкеры**: предметы внутри шалкеров тоже учитываются.
- Подсчёт **полных шалкеров**: если шалкер заполнен одним видом предмета и
  количество = `1728` (стак 64) / `432` (стак 16), он засчитывается как полный
  шалкер этого предмета.
- Поддержка форматов предметов до и после 1.20.5 (`tag.BlockEntityTag` и `components`).

## Установка

```bash
npm install
```

## Настройка

Правится в [`config.js`](config.js):

```js
export const worlds = [
  {
    name: "world",
    regionDir: "./worlds/world/region",
    dimensions: [
      {
        name: "Склад",
        pos1: { x: 100, y: -60, z: 100 },
        pos2: { x: 140, y: 20, z: 140 },
      },
    ],
  },
];
```

`pos1`/`pos2` — два противоположных угла кубоида (порядок не важен).

## Запуск

```bash
node index.js
```

Результат — `containers.json` со структурой: миры → кубоиды, где на каждом уровне
(кубоид → мир → весь сервер) лежат агрегаты:

- `totals` — количество каждого предмета (`id` → кол-во), с учётом содержимого шалкеров;
- `fullShulkers` — количество полных шалкеров каждого предмета.

Координаты отдельных контейнеров в файл не выводятся — только суммарные количества.

## Запуск на Ubuntu

### 1. Установить Node.js (18+)

```bash
# из официального репозитория NodeSource (рекомендуется)
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs git

# проверка
node -v
npm -v
```

### 2. Клонировать репозиторий и установить зависимости

```bash
git clone https://github.com/IronCAT09/MinecraftConteinerScanner.git
cd MinecraftConteinerScanner
npm install
```

### 3. Указать пути к регионам

Регионы лежат в каталоге мира на сервере, например:

```
<сервер>/world/region/r.X.Z.mca           # обычный мир
<сервер>/world_nether/DIM-1/region/...     # ад
<сервер>/world_the_end/DIM1/region/...     # энд
```

Пропиши абсолютные пути прямо в `config.js` (`regionDir`), либо положи/смонтируй
регионы рядом. Скрипт читает файлы только на чтение — копировать не обязательно,
достаточно прав на чтение.

### 4. Запустить

```bash
node index.js
```

Появится `containers.json` в текущем каталоге.

### Автообновление по расписанию (опционально)

Чтобы JSON для сайта обновлялся, например, каждые 10 минут, через `cron`:

```bash
crontab -e
```

```cron
*/10 * * * * cd /path/to/MinecraftConteinerScanner && /usr/bin/node index.js >> scan.log 2>&1
```

> Регионы на диске обновляются не мгновенно — Minecraft сбрасывает чанки на диск
> при выгрузке/автосохранении. Для актуальных данных запускай сканер после
> сохранения мира (или по нему).

## Структура

- [`index.js`](index.js) — обход миров/кубоидов, сбор предметов, полные шалкеры.
- [`region.js`](region.js) — чтение Anvil-формата `.mca` (заголовок + распаковка чанков).
- [`config.js`](config.js) — миры, кубоиды, пути.
